## Security Recommendations

This repository was created with `visibility: private` by default. For production workflows, we recommend enabling the following GitHub native features via Settings:

- **[Branch protection rules](https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/managing-protected-branches/about-protected-branches)** — require PR + review + status checks, disallow force-push on `main`
- **[Dependabot alerts](https://docs.github.com/en/code-security/dependabot/dependabot-alerts/about-dependabot-alerts)** — weekly scan for vulnerable dependencies
- **[Secret Scanning](https://docs.github.com/en/code-security/secret-scanning/about-secret-scanning)** — auto-detect committed secrets (free for private repos since 2023)
- **[Tag protection rules](https://docs.github.com/en/repositories/configuring-branches-and-merges-in-your-repository/configuring-tag-protection-rules)** — restrict who can push `v*` tags (controls production deploys)

Fluxomind does not enforce these — they are governance decisions of your team via GitHub Settings.
